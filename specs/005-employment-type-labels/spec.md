# Feature Specification: 雇用区分の表示ラベル変更

**Feature Branch**: `005-employment-type-labels`  
**Created**: 2025-11-19  
**Status**: Draft  
**Input**: User description: "雇用区分の種類を以下のように修正してください。FULL_TIME - 常勤、PART_TIME - パートタイム、CONTRACT - 契約社員"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 雇用区分の表示ラベル変更 (Priority: P1)

統括人事管理者または現場マネージャーが従業員登録・編集画面、従業員一覧画面、従業員詳細画面を表示した際、雇用区分の表示ラベルが「FULL_TIME - 常勤」「PART_TIME - パートタイム」「CONTRACT - 契約社員」として表示される。

**Why this priority**: 雇用区分の表示ラベルは、ユーザーが従業員情報を理解するための重要な情報です。FULL_TIMEの表示を「正社員」から「常勤」に変更することで、より正確な表現となり、ユーザーの理解を向上させます。

**Independent Test**: 統括人事管理者が従業員登録画面を表示し、雇用区分の選択肢に「常勤」「パートタイム」「契約社員」が表示されることを確認します。この機能単体で、雇用区分の表示ラベルが正しく変更されたことを検証できます。

**Acceptance Scenarios**:

1. **Given** 統括人事管理者が従業員登録画面を表示している, **When** 雇用区分の選択肢を確認する, **Then** 「常勤」「パートタイム」「契約社員」の3つの選択肢が表示される
2. **Given** 統括人事管理者が従業員編集画面を表示している, **When** 既存の従業員の雇用区分を確認する, **Then** FULL_TIMEの従業員には「常勤」、PART_TIMEの従業員には「パートタイム」、CONTRACTの従業員には「契約社員」が表示される
3. **Given** 統括人事管理者が従業員一覧画面を表示している, **When** 各従業員の雇用区分を確認する, **Then** FULL_TIMEの従業員には「常勤」、PART_TIMEの従業員には「パートタイム」、CONTRACTの従業員には「契約社員」が表示される
4. **Given** 統括人事管理者が従業員詳細画面を表示している, **When** 従業員の雇用区分を確認する, **Then** FULL_TIMEの従業員には「常勤」、PART_TIMEの従業員には「パートタイム」、CONTRACTの従業員には「契約社員」が表示される
5. **Given** 統括人事管理者が従業員登録画面で新規従業員を登録する, **When** 雇用区分として「常勤」を選択して保存する, **Then** データベースには「FULL_TIME」として保存され、画面には「常勤」として表示される

---

### Edge Cases

- 既存のデータベースに保存されている雇用区分の値（FULL_TIME, PART_TIME, CONTRACT）は変更されない
- 雇用区分のフィルタリング機能（一覧画面での検索・フィルタ）は、データベースの値（FULL_TIME, PART_TIME, CONTRACT）を基準に動作し、表示ラベルの変更の影響を受けない
- CSVエクスポート機能では、データベースの値（FULL_TIME, PART_TIME, CONTRACT）が出力される（表示ラベルの変更の影響を受けない）
- 既存のAPIレスポンスでは、データベースの値（FULL_TIME, PART_TIME, CONTRACT）が返される（表示ラベルの変更の影響を受けない）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは従業員登録・編集画面の雇用区分選択肢で、「常勤」「パートタイム」「契約社員」の3つのラベルを表示すること。データベースの値（FULL_TIME, PART_TIME, CONTRACT）との対応は以下の通りとする：
  - FULL_TIME → 「常勤」
  - PART_TIME → 「パートタイム」
  - CONTRACT → 「契約社員」
- **FR-002**: システムは従業員一覧画面で、各従業員の雇用区分を「常勤」「パートタイム」「契約社員」のいずれかで表示すること。
- **FR-003**: システムは従業員詳細画面で、従業員の雇用区分を「常勤」「パートタイム」「契約社員」のいずれかで表示すること。
- **FR-004**: システムは雇用区分のフィルタリング機能（一覧画面での検索・フィルタ）が、データベースの値（FULL_TIME, PART_TIME, CONTRACT）を基準に動作し続けること（表示ラベルの変更の影響を受けないこと）。
- **FR-005**: システムはCSVエクスポート機能で、データベースの値（FULL_TIME, PART_TIME, CONTRACT）を出力すること（表示ラベルの変更の影響を受けないこと）。
- **FR-006**: システムは既存のAPIレスポンスで、データベースの値（FULL_TIME, PART_TIME, CONTRACT）を返すこと（表示ラベルの変更の影響を受けないこと）。
- **FR-007**: システムはデータベースに保存されている雇用区分の値（FULL_TIME, PART_TIME, CONTRACT）を変更しないこと。

### Key Entities *(include if feature involves data)*

- **従業員（Employee）**: 雇用区分（employment_type）フィールドを含む。データベースには「FULL_TIME」「PART_TIME」「CONTRACT」のいずれかの値が保存される。表示時には以下のラベルに変換される：
  - FULL_TIME → 「常勤」
  - PART_TIME → 「パートタイム」
  - CONTRACT → 「契約社員」

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 従業員登録・編集画面で、雇用区分の選択肢に「常勤」「パートタイム」「契約社員」が100%の精度で表示されること
- **SC-002**: 従業員一覧画面で、各従業員の雇用区分が「常勤」「パートタイム」「契約社員」のいずれかで100%の精度で表示されること
- **SC-003**: 従業員詳細画面で、従業員の雇用区分が「常勤」「パートタイム」「契約社員」のいずれかで100%の精度で表示されること
- **SC-004**: データベースに保存されている雇用区分の値（FULL_TIME, PART_TIME, CONTRACT）が変更されないこと（100%の精度）
- **SC-005**: 雇用区分のフィルタリング機能が、データベースの値（FULL_TIME, PART_TIME, CONTRACT）を基準に正しく動作すること（100%の精度）
- **SC-006**: CSVエクスポート機能で、データベースの値（FULL_TIME, PART_TIME, CONTRACT）が正しく出力されること（100%の精度）
- **SC-007**: 既存のAPIレスポンスで、データベースの値（FULL_TIME, PART_TIME, CONTRACT）が正しく返されること（100%の精度）

## UI要件

- **UI-001**: システムは雇用区分の表示ラベルを、すべての画面（登録・編集・一覧・詳細）で一貫して「常勤」「パートタイム」「契約社員」として表示すること。
- **UI-002**: システムは雇用区分の選択肢（ドロップダウン、ラジオボタンなど）で、「常勤」「パートタイム」「契約社員」の3つの選択肢を表示すること。

## 前提条件

- データベースに雇用区分（employment_type）フィールドが存在し、FULL_TIME, PART_TIME, CONTRACTの3つの値が定義されていること（既存のスキーマ定義に準拠）
- 既存の従業員データがデータベースに保存されていること
- システムは既存の従業員管理機能が動作していること

## 依存関係

- 既存の従業員管理機能（001-employee-db-requirements）
- データベーススキーマ（database/supabase_schema.sql）の雇用区分制約定義

## 対象外

- データベースの値（FULL_TIME, PART_TIME, CONTRACT）の変更（表示ラベルの変更のみ）
- 雇用区分の種類の追加・削除（既存の3種類の表示ラベルの変更のみ）
- CSVエクスポート時の表示ラベル変換（データベースの値をそのまま出力）
- APIレスポンスの表示ラベル変換（データベースの値をそのまま返す）
